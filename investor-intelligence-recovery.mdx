# Investor Intelligence Recovery

<Info>
Investor-intelligence is an active completion program, not a finished surface. Use this page to understand what is already live, what is still partial, and which routes are being elevated into the one-API investor-agent contract.
</Info>

## Current posture

- `SEC`, ownership, filing traces, and normalized statement workflows are production-grade
- market, macro, factor, and semantic-intelligence expansion is partially live but still closing material substrate and cutover gaps
- public docs for the investor-intelligence surface are being published in lockstep with the recovery program

## Recovery goals

<Steps>
  <Step title="One API for investor agents">
    Agents should be able to use one auth model, one SDK family, one MCP surface, and one semantic query surface for investor workflows.
  </Step>
  <Step title="Fast, compact answers">
    The target posture is sub-100ms standard investor queries and hot bundles that stay compact enough to be dropped directly into an agent turn.
  </Step>
  <Step title="Traceable recommendations">
    Every factor, macro, filing, and hedge recommendation must carry provenance, freshness, methodology, and rights metadata.
  </Step>
</Steps>

## What to trust today

Use the existing SEC-native routes and guides for:

- issuer research
- filing extraction
- ownership workflows
- compensation and governance analysis
- trace hydration

Treat the broader investor-intelligence surface as `expanding` until the following are fully documented and benchmarked:

- market reference and estimates
- tier-1 macro coverage and revisions
- factor returns, exposures, and strategy routes
- semantic investor query workflows
- portfolio stress-test and optimization flows
