# GET /v1/statements/segmented-revenues

Return filing-derived segmented revenue history with XBRL product or geography dimensions, capability state, and trace references when issuers disclose them

<Info>
Audience: application and coding agent.
</Info>

## Query parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `ticker` | string | Yes (or `cik`) | Ticker symbol (e.g., `AAPL`, `PFE`) |
| `cik` | string | Yes (or `ticker`) | SEC CIK number |
| `segment_type` | string | No | `geographic`, `product`, or `other`. Filters to the specified segment dimension. |
| `period` | string | No | `annual` (default) or `quarterly`. Controls which filing forms are scanned (10-K vs 10-Q). |
| `limit` | integer | No | Number of filing periods to return (default 6, max 50). |
| `submission_file_limit` | integer | No | Number of SEC submission history files to scan for older filings (default 8, max 50). Increase for deep historical queries. |

## Canonical metadata

- `requestId`
- `traceparent`
- `provenance`
- `freshness`
- `materialization`

## Key response fields

| Field | Description |
|-------|-------------|
| `records[].segmentLabel` | Human-readable segment name (e.g., "Developed Europe", "EMEA") |
| `records[].segmentType` | `geographic`, `product`, or `other` |
| `records[].hierarchyDepth` | Depth in the segment hierarchy (1 = top-level, 2 = regional) |
| `records[].isMostGranularSibling` | `true` for leaf segments suitable for analysis; `false` for parent containers |
| `records[].value` | Revenue value in the reported currency |
| `records[].periodEnd` | End date of the reporting period |

## Example request

<RequestExample>
```bash
curl -X GET \

```
